# [크롤러 수정 요청] 차량 vehicle_specs 모델 파싱 — 감정서 스캔본 대응

작성: 2026-07-08 · 대상: 스피드옥션 크롤러(vehicle_specs 적재 로직)

---

## 1. 한 줄 요약
차량 경매물건 일부가 `vehicle_specs.model`에 **실제 모델명 대신 차종 카테고리('SUV'·'승용자동차')**가 들어가고, **나머지 제원(제조사·연식·연료·배기량·주행거리·색상)이 전부 NULL**로 적재됨 → 앱에서 엔카 시세 매칭 불가.

## 2. 규모 (Supabase `vehicle_specs`)
- 전체 8,479건 중 **model이 차종 카테고리 = 122건** (`승용자동차` 75 · `SUV` 47).
- 그중 현재 진행중(items.data_class='현황') 차량 **86건** → 목록에서 시세 안 나옴.

## 3. 증거 (예시 3건)
| item_key | 실제 차 (items.usage_name) | 현재 vehicle_specs 저장값 |
|---|---|---|
| `I01\|2026\|100436\|1` | 캐스퍼( CASPER)현대자동차 | 제조사=NULL · **model='SUV'** · 연식·연료·배기량·주행·색 전부 NULL |
| `E01\|2026\|81\|1` | GV80현대자동차(주) | 제조사=NULL · **model='SUV'** · 나머지 NULL |
| `E01\|2026\|50904\|1` | XM3르노삼성코리아 | 제조사=NULL · **model='SUV'** · 나머지 NULL |

→ **실제 모델은 물건명(usage_name)에 멀쩡히 있음.** vehicle_specs만 비어 있음.

## 4. 근본 원인 (실측 확인됨)
- 이 차들의 **감정평가서 PDF가 스캔 이미지본** → 텍스트 추출 시 **16~21자**(사실상 빈 문서). 크롤러의 텍스트 파싱이 제원표(제조사·모델·연식·배기량·주행)를 못 뽑음.
- **매각물건명세서**는 텍스트 있음(1,500~1,800자)이고 **차대번호는 있으나 배기량 등 제원표는 불완전**(배기량 미포함).
- 그 결과 크롤러가 차종 카테고리('SUV')만 model에 넣고 나머지는 NULL 처리.

## 5. 영향 (앱 측)
- 앱은 엔카 중고차 시세를 **제조사·모델·연식·주행거리**로 매칭하는데 이 필드가 없어 매칭 불가.
- 'SUV'만으로 매칭하면 아무 SUV(벤츠 EQE 등)에 붙어 **캐스퍼가 8,305만원** 같은 엉터리 시세가 남 → 앱은 임시로 **분류어(SUV·승용 등) 매칭을 차단**(시세 미표시)해 둠. **크롤러가 model을 제대로 넣으면 자동 복구됨.**

## 6. 수정 요청 (우선순위 순)
1. **감정평가서 OCR**: 스캔본 감정서를 OCR → 제원표(제조사·실모델·연식·연료·배기량·주행거리) 추출 후 vehicle_specs 채우기. ← 가장 완전.
2. **물건명에서 실모델 추출(최소 조치)**: 최소한 `model`에 차종('SUV') 대신 **실제 모델명**을 넣기. `usage_name`이 `"모델명+제조사"` 형태(예: `캐스퍼( CASPER)현대자동차`, `GV80현대자동차(주)`, `XM3르노삼성코리아`) → 제조사 꼬리 제거하면 모델. (연식·주행은 OCR 필요)
3. **명세서 '자동차의 표시' 파싱 강화**: 명세서에 차대번호는 있으니 제원표 영역을 더 파싱.

※ 폴백 규칙 제안: 감정서 텍스트가 비정상적으로 짧으면(예: <100자 = 스캔본) 감정서 파싱을 신뢰하지 말고 ①OCR ②물건명 모델 ③명세서 순으로 보완. 그리고 **model에는 절대 차종 카테고리(SUV/승용자동차/화물자동차 등)를 넣지 않기.**

## 7. 검증 방법
수정 후 아래 카운트가 0에 가까워야 함:
```sql
SELECT count(*) FROM vehicle_specs
WHERE model IN ('SUV','RV','승용','승용차','승용자동차','승합','승합차','승합자동차',
                '화물','화물차','화물자동차','특수','자동차','중기','건설기계',
                '세단','해치백','쿠페','왜건','밴','트럭','버스','경형','소형','중형','대형');
```

## 8. 참고 — 앱 측은 이미 조치 완료
- A: 분류어 매칭 차단(엉터리 시세 방지) · B: 시세 캐시 전량 갱신(16→499대) · C: 일괄매각 표시.
- **크롤러가 model/제원을 제대로 적재하면**, 앱의 다음 시세 갱신(`warm_encar.py` 재실행) 때 이 차들도 자동으로 정상 시세가 뜸. 앱 코드 추가 수정 불필요.
