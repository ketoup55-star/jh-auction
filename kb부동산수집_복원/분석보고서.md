# kb+부동산+수집.exe 역분석 보고서

## 1. 패키징 / 빌드 정보
- **빌더**: PyInstaller (onefile), **Python 3.13**, Windows x64
- **파일 크기**: 365 MB — Playwright(Chromium 브라우저) 번들 포함이 주원인
- **원본 패키지 경로**: `kmong/kb2/` (import 경로 `kmong.kb2.lawd_data`로 확인)
  - "kmong"은 외주 플랫폼(크몽) 작업물 네임스페이스로 추정
- **추출 방법**: PyInstaller 아카이브 추출 → `.pyc` → 3.13 디스어셈블/역분석

## 2. 프로그램 개요
KB부동산(kbland.kr)에서 **지역·물건종류·거래유형별 매물 목록을 수집해 엑셀로 저장**하는
데스크톱 GUI 프로그램. 카카오 로그인으로 API 인증 헤더를 획득한 뒤 KB Land 내부 API를 직접 호출한다.

## 3. 모듈 구성 (사용자 코드 5개)
| 모듈 | 역할 |
|------|------|
| `ui_app.py` | customtkinter GUI (메인). 지역 선택·필터·수집 실행·엑셀 저장 |
| `fetch.py` | **핵심 크롤러**. Playwright 로그인 + KB Land API 호출/파싱 |
| `service.py` | 시도/시군구/읍면동 이름·코드 조회 |
| `lawd_data.py` | 법정동코드 데이터셋(`LAWD_CD_LIST`) — 로직 아님, 순수 데이터 |
| `kmong.py` | 빈 패키지 마커 (내용 없음) |

> 나머지 ~1,140개 모듈은 표준 라이브러리 + 서드파티(playwright, requests, customtkinter,
> openpyxl, numpy, PIL, bs4 등)이며 사용자 코드가 아님.

## 4. 동작 흐름 (fetch.py)
1. **로그인** `login(email, password)`
   - Playwright(Chromium)로 `https://kbland.kr/` 접속 → 홈 팝업 닫기 → 로그인 다이얼로그 열기
   - `.btn.btn-login.kakao` 클릭 → 카카오 로그인 팝업에서 `loginId`/`password` 입력 후 submit
   - `context.route()`로 `^https://api\.kbland\.kr/` 요청을 가로채 **pointInfo API 호출의 request 헤더를 캡처**
     (이 헤더가 이후 모든 API 호출의 인증 수단)
   - 헤더가 안 잡히면 `https://kbland.kr/al?xy=37.5665,126.9780,16` 지도 URL로 강제 트리거
2. **좌표 변환** `get_lat_lng(주소)`
   - `GET https://api.kbland.kr/land-complex/serch/intgraSerch`
   - params: `검색설정명=SRC_JUSO, 검색키워드=<주소>, 출력갯수=2, 페이지설정값=1`
   - 응답의 `dataBody.data.data.JUSO.data[0].WGS84_LAT/LNG` 추출
3. **매물 수집** `get_all_properties(...)`
   - `POST https://api.kbland.kr/land-property/propList/stutCdFilter`
   - 1페이지=30건, 총건수(`총매물건수`)로 페이지 수 계산 후 전체 순회
   - 실패 시 최대 3회 재시도(`requests.RequestException`)
   - payload 주요 키: `법정동코드`(10자리 fullCode), `물건종류`, `거래유형`,
     `startLat/Lng`·`endLat/Lng`(동일 좌표), `페이지번호`, `페이지목록수`, 그 외 다수 필터(빈값)

### 코드 매핑(main 기본값 기준)
- `물건종류 = "16,19,21,23,20,22,28,43"` (콤마구분 다중)
- `거래유형 = "1,2,3"` (매매/전세/월세 추정)
- `법정동코드 = "3017010300"` (대전 서구 도마동)
